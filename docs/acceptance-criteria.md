## 채워야 할 시나리오 (번호는 `feature_list.json`과 일치해야 함)

1. (협업) 두 클라이언트가 같은 room에 접속해 캔버스 오브젝트를 함께 편집
2. (협업) A가 오브젝트를 이동하면 B의 화면에도 실시간 반영
3. (협업) 동시 편집 충돌 시 Yjs CRDT 병합 동작 확인
4. (협업) 한 클라이언트가 재접속해도 상태 동기화 유지
5. (협업) awareness(커서/선택 상태) 공유 확인
6. (생성+그래프) 프롬프트 입력 → 이미지 생성 → 노드 그래프에 새 노드 추가
7. (생성+그래프) 기존 노드에서 분기 생성 → 엣지로 연결
8. (생성+그래프) 노드 그래프 탐색 시 캔버스에 반영
9. (3D) 캔버스 합성 결과가 3D 모델 텍스처로 반영, OrbitControls로 회전 확인
10. (생성) 생성 결과를 캔버스 오브젝트로 추가
11. (생성) 생성 실패 시 에러 처리 확인

## 각 시나리오 작성 시 포함할 것

- 사전 조건
- 조작 순서
- 기대 결과 (사람이 눈으로 확인 가능한 수준으로 구체적으로)
- 자동화 가능 여부 (Vitest/Supertest로 자동화할 수 있는지, 수동 확인이 필요한지)
